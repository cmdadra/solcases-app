üîê How It Works
1. Server Seed
A future Solana block is selected before each pack opening. Once the block is mined, its hash is used as the server seed, ensuring results cannot be manipulated.
2. Client Seed
This is a unique seed provided by your wallet, which can be customized in your account settings for additional randomness.
3. Nonce
A counter that increments with every pack opened, ensuring each result is completely unique and cannot be repeated.
‚úÖ Verification
Players can replicate any past pack opening by using the code provided below, or verify it using external tools. Every result is mathematically provable and cannot be rigged.
Our system uses SHA-512 hashing and is completely transparent. You can verify every single pack opening to ensure fairness.
üíª Verification Code
const crypto = require("crypto");
const serverSeed = "example_server_seed";
const clientSeed = "example_client_seed";
const nonce = 1;
const min = 0;
const max = (100*10000) -1;

const hash = crypto.createHash('sha512').update(`${serverSeed}-${clientSeed}-${nonce}`).digest('hex');
const ticket = seededRandomInteger(hash, min, max);
console.log(`Generated ticket: ${ticket}`);

function seededRandomInteger(hash, min, max) {
  const range = max - min + 1;
  const uintValue = parseInt(hash.substring(0, 8), 16);
  return min + (uintValue % range);
}
